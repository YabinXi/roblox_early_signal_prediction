# AutoResearch — Research Program

## Research Question
**在 Roblox 平台历史数据中，中腰部游戏（排名50-200）出现"DAU偏低但D7留存/平均时长显著高于同品类均值"的异常信号后，该品类在随后3-6个月内产生Top 10爆款的概率是多少？该信号的精确率(Precision)和召回率(Recall)分别是多少？**

## Usage
This file is the meta-instruction for the orchestrator (Claude Code).
When Claude Code reads this file, it should autonomously execute the research
iteration loop described below.

## Mandatory Research Integrity Rules

Before any analysis, review and enforce these 6 rules (from SKILL.md):

1. **R1: Verify Before Analyze** — `data_audit.json` + user confirmation gate
2. **R2: Declare Data Window Limitations** — `temporal_limitation` in every output
3. **R3: No Vague Conclusions** — exact numbers, dates, effect sizes, CIs
4. **R4: Enumerate Confounders** — ≥3 per comparison, with direction + control status
5. **R5: Clean Window Default** — use clean baseline periods, not arbitrary pre-periods
6. **R6: Decompose Growth** — incremental vs cannibalization for any growth claim

## Autonomous Iteration Loop

### Pre-flight Checks
1. Confirm `data/data_snapshot.json` exists
   - If missing: `uv run python prepare.py`
2. Confirm `outputs/findings.json` exists
   - If missing: `uv run python analyze.py`
3. Confirm `outputs/report.md` exists
   - If missing: `uv run python report.py`

### Each Iteration

#### Phase 1: Read Feedback
```bash
cat outputs/eval_result.json
```
Extract `key_gaps`, `priority_improvements`, dimension scores.

#### Phase 2: Improve Analysis (Analyst role)
1. Read `agent_prompts/analyst.md`
2. Choose improvement strategy based on lowest-scoring dimension
3. Edit `analyze.py`
4. Run: `uv run python analyze.py`

#### Phase 3: Improve Report (Writer role)
1. Read `agent_prompts/writer.md`
2. Edit `report.py` based on feedback
3. Run: `uv run python report.py`

#### Phase 4: Evaluate (Reviewer role)
1. Run: `uv run python evaluate.py`
2. Read `outputs/eval_result.json`

#### Phase 5: Git Decision
- **Score improved** → `git commit`
- **Score declined** → `git checkout --` to restore

### Fast-Cycle Mode
Every 3 full iterations, run Phase 2 only, using findings.json self-check
metrics for quick validation.

### Stop Conditions
- `total_score >= 80` — quality target met
- 3 consecutive iterations with Δ ≤ ±2 — converged
- 20 total iterations — cap reached

## Scoring Dimensions

| Dimension | Weight | Focus |
|---|---|---|
| Data Support | 25% | Sources, tests, sample sizes |
| Logical Rigor | 20% | Causal inference, confounders, clean windows |
| Insight Depth | 20% | Beyond descriptive; industry/behavioral logic |
| Hypothesis Coverage | 15% | Systematic enumeration, decomposition |
| External Validity | 10% | Generalizability, cross-validation |
| Actionability | 10% | Decision guidance, quantified predictions |
